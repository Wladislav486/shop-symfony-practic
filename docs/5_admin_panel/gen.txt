
030 Страница авторизации
1 Создаём контроллер авторизации админки в отдельной папке admin/
2 Создаём представление, также выносим в отдельную папку
3 Подключаем стили и скрипты для админки в base.html.twig
4 Настраиваем отдельную систему авторизации для админки для этого правим
app/config/packages/security.yaml
5 создаём app/src/Security/Admin/LoginFormAuthenticator.php
- класс реализующий логику авторизации регистрации



031 Проверка на роль ROLE_ADMIN
1 Правим фун-ал регистрации авторизации
- при проверке прользователя на доступ к ресурсу проверяем дополнительно его роль
2 Добавляем одному из пользователей роль админ
3 проверяем на админе и не админе работу фун-ала


032 Главная страница раздела admin
1 Создаём контроллер который подключает представление и реализуем в нём роутинг
2 Создаём шаблоны для страницы
3 Вносим правки в фун-ал авторизации/регистрации
- при успешной регистрации реализуем редирект на главную страницу админки

033 Ограничение доступа
- в админку смогут войти только пользователи с ролью ROLE_ADMIN

034 Команда AddUserCommand
- команда консоли добавляюшщая новых пользователей определённой роли
1 С помощью Symfony CLI создаём класс команды
2 Корректируем параметры, проверяем вывод входных данных
3 Настраиваем взаимодействие из консоли с фун-алом
- реализуем валидацию значений
- задаём для роли админа дефолтное значений
4 Реализуем добавление пользователя в бд
5 Пишем сообщения для вывода в консоль при суспешном добавлении и при ошибке
